All tests should be written to FAIL initially (RED phase). They define the expected behavior that Plans 01-01 and 01-02 must satisfy.

**tests/test_bridge_output.py** — JSON output contract tests:

1. `test_output_success_returns_valid_json()` — imports `output_success` from common, captures stdout, asserts output is valid JSON.
2. `test_output_error_returns_error_json()` — imports `output_error` from common, captures stdout (use capsys or redirect), asserts JSON contains `error: true`, `code`, `message`, `retryable` keys.
3. `test_output_error_exits_with_code_1()` — asserts `output_error` calls `sys.exit(1)`.
4. `test_all_bridge_scripts_importable(scripts_dir)` — for each of the 5 bridge scripts (twitter_search, tweet_scorer, reply_generator, twitter_post, report_generator), attempts `importlib.import_module()` after adding scripts_dir to sys.path. Asserts no ImportError.

**tests/test_error_handling.py** — Error classification tests:

1. `test_classify_rate_limit_error()` — creates a mock httpx.HTTPStatusError with status 429, passes to `classify_error()`, asserts returns `("RATE_LIMITED", True)`.
2. `test_classify_service_unavailable()` — mock 503 error, asserts `("SERVICE_UNAVAILABLE", True)`.
3. `test_classify_timeout()` — mock httpx.TimeoutException, asserts `("TIMEOUT", True)`.
4. `test_classify_cloudflare_block()` — Exception with "Cloudflare" in message, asserts `("CLOUDFLARE_BLOCK", True)`.
5. `test_classify_auth_failure()` — Exception with "cookie" in message, asserts `("TWIKIT_AUTH_FAILED", False)`.
6. `test_classify_unknown_error()` — generic Exception, asserts `("UNKNOWN_ERROR", False)`.

**tests/test_ops_logging.py** — Operational logging tests:

1. `test_log_action_inserts_row(initialized_db)` — calls `log_action("test_action", True)`, queries agent_actions table, asserts 1 row with correct action_type and success=1.
2. `test_log_action_records_error_code(initialized_db)` — calls `log_action("test_fail", False, error_code="TEST_ERR")`, asserts row has error_code="TEST_ERR".
3. `test_log_action_records_cost(initialized_db)` — calls `log_action("test_cost", True, tokens_used=100, cost_usd=0.001)`, asserts row has correct values.
4. `test_log_action_does_not_crash_on_db_error()` — temporarily sets MEMORY_DB to an invalid path, calls log_action, asserts it does NOT raise (logs to stderr instead).

**tests/test_memory_schema.py** — Memory schema tests:

1. `test_scored_tweets_table_exists(initialized_db)` — queries sqlite_master, asserts scored_tweets table exists.
2. `test_scored_tweets_has_17_columns(initialized_db)` — uses PRAGMA table_info, asserts 17 columns.
3. `test_scored_tweets_column_names(initialized_db)` — asserts exact column names: id, tweet_id, text, username, bio, follower_count, verified, score, category, reason, should_reply, opportunity_summary, found_at, query_used, replied, reply_text, replied_at.
4. `test_agent_actions_table_exists(initialized_db)` — asserts agent_actions table exists.
5. `test_agent_actions_has_8_columns(initialized_db)` — asserts 8 columns.
6. `test_scored_tweets_unique_tweet_id(initialized_db)` — inserts a tweet, inserts duplicate tweet_id, asserts IntegrityError.
7. `test_init_db_creates_directory()` — verifies init_db creates the memory directory if missing.

Each test file should have a module docstring explaining what it validates. Tests should use descriptive names.
